Program parameters

AFF3CT is a command line style program which runs over a standard console. This section describes the various parameters you can play with. Depending on the type of simulation you want to run, some arguments can be available or unavailable. Full command line examples are given in the Running examples section.

Common parameters

Parameters Values Description
--code-type POLAR, TURBO, RA, REPETITION, RSC The family of error correction codes to be simulated.
--simu-type BFER, EXIT The type of simulation. It can be either a frame and bit error rate simulation or an EXIT chart simulation.
--version AFF3CT software version.
--help, -h Display the help.
-K Integer The number of information bits per frame.
-N Integer The frame length (information bits + redundancy bits).
--snr-min Float Minimum value of Signal over Noise Ratio to be simulated (in dB).
--snr-max Float Maximum value of Signal over Noise Ratio to be simulated (in dB).
--snr-step Float Step value of Signal over Noise Ratio to be simulated (in dB).
--disable-display Add this parameter to inhibit console output.
--prec 8, 16, 32, 64 Format of the bits and LLRs (number of bits).
--qn-bits Integer Dynamic of the LLRs (available when --prec is set to 8 or 16).
--qpoint-pos Integer Number of fractional bits (fixed point logic).
--n-threads Integer Number of parallel threads to be launched.
--code-gen-method RAND, LCG, AZCW Statistical method to generate input frames (RAND: STL C++ library ; LCG: Linear Congruential Generator ; AZCW: All Zero CodeWord).
--channel-type NO, AWGN, AWGN_GSL The type of channel to be used in the simulation (NO: No channel, AWGN: STL normal distribution ; AWGN_GSL: GNU Scientific Library normal distribution).
--stop-time Integer The limit of simulation time for each SNR value (in seconds).

BFER simulations parameters

Parameters Values Description
--max-fe Integer Number of simulated frame errors.
--enable-debug Enable debugging output, displaying intermediate values taken by binary and real vectors in the simulation chain.

EXIT chart simulations parameters

For further informations about EXIT charts, please see the academic paper from Stephan ten Brink, Convergence Behavior of Iteratively Decoded Parallel Concatenated Codes.

Parameters Values Description
--sig-a-min Float Minimum value of σA, which is a parameter for the Gaussian model of a priori input in EXIT charts.
--sig-a-max Float Maximum value of σA, which is a parameter for the Gaussian model of a priori input in EXIT charts.
--sig-a-step Float Step value of σA, which is a parameter for the Gaussian model of a priori input in EXIT charts.

Polar codes parameters

Parameters Values Description
--disable-sys-enc Disable systematic encoding, which is enabled by default.
-L Integer Set the list depth in SC List decoders.
--dec-algo SC, SCL, SCAN Choose decoding algorithm of the polar codes (SC: Successive Cancellation ; SCL: Successive Cancellation List ; SCAN : Soft Cancellation).
--dec-implem NAIVE, FAST Choose decoding implementation of the polar codes (NAIVE is the basic version, FAST is an optimized version).
--dec-simd-strat INTRA, INTER Choose the SIMD strategy to adopt in the decoding. Vectorization increases the decoding throughput. (INTRA: SIMD is used to speed up the decoding of a frame, INTER: SIMD is used to decode multiple frames simultaneously)
--code-sigma Float Specify a fixed value for the frozen bits generation.
--awgn-codes-dir Path to directory Choose a directory to store and read frozen bits files.
--awgn-codes-file Path to file Choose a file to use fixed frozen bits file.
--crc-type CRC-32-GZIP, CRC-24-LTEA, CRC-16-IBM, CRC-16-CCITT, CRC-8-DVB-S2, CRC-4-ITU, CRC-3-0x3, CRC-2-0x2, CRC-1-0x1 Concatenate a cyclic redundancy check to the error correcting code (used in the SCL decoder).

Turbo codes parameters

Parameters Values Description
--dec-algo BCJR Choose decoding algorithm of the inner RSC codes (BCJR: Bahl-Cocke-Jelinek-Raviv algorithm).
--dec-implem STD, FAST, VERY_FAST Different implementations of the algorithm (STD: naive implementation, FAST and VERY FAST: optimized versions).
--dec-simd-strat INTRA, INTER Choose the SIMD strategy to adopt in the decoding. Vectorization increases the decoding throughput. (INTRA: SIMD is used to speed up the decoding of a frame, INTER: SIMD is used to decode multiple frames simultaneously)
--interleaver NO, LTE, RANDOM Choose the interleaver for the iterative process (NO: no interleaver, LTE : LTE standard interleaver, RANDOM : random interleaver (constant during simulation)).
--scaling-factor NO, LTE, LTE_VEC, ARRAY, FLOAT Choose the scaling factor for the iterative process (NO: no scaling factor, LTE: scaling factor is 0.75, LTE_VEC: optimized version of LTE scaling factor, ARRAY: custom scaling factor (changing over iterations), FLOAT: custom constant value).
--crc-type CRC-32-GZIP, CRC-24-LTEA, CRC-16-IBM, CRC-16-CCITT, CRC-8-DVB-S2, CRC-4-ITU, CRC-3-0x3, CRC-2-0x2, CRC-1-0x1 Concatenate a cyclic redundancy check to the error correcting code.
